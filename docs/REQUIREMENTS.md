# 요구사항 명세서

## 1. 회원 로그인

| 항목 | 내용 |
|------|------|
| 기능 | 환자 및 의사가 ID / Password로 인증 후 로그인 |
| 회원 유형 | 환자(Patient), 의사(Doctor) |
| 데이터 | 회원 테이블 및 초기 데이터 제공 |

**요구사항**
- 로그인 성공 시 회원 유형(환자/의사)에 따라 접근 권한을 구분한다.
- 인증 실패 시 적절한 오류 메시지를 표시한다.

---

## 2. 회원 목록 조회

| 항목 | 내용 |
|------|------|
| 기능 | 회원 목록 표시 및 검색 |

**요구사항**
- **환자**: 자기 자신의 정보만 조회 가능
- **의사**: 모든 환자의 목록 조회 가능
- 이름, ID 등 조건으로 검색 기능 제공

---

## 3. 회원 상세 조회

| 항목 | 내용 |
|------|------|
| 기능 | 개별 회원의 상세 정보 및 실시간 건강 데이터 그래프 표시 |
| 데이터 | 질병코드 테이블 및 초기 데이터 제공 |

**표시 항목**
- 회원 ID
- 이름
- 생년월일
- 성별
- 보유 질병 정보 (질병코드 기반)
- 메모

**실시간 그래프**
- 심박수
- 혈당
- 걸음수

---

## 4. 실시간 건강정보 조회

건강정보 제공 서버(WebSocket): `https://healthsim.iranglab.com`

### 4-1. 심박 정보

| 필드 | 설명 |
|------|------|
| 회원 ID | 측정 대상 회원 식별자 |
| 측정일시 | 측정이 이루어진 일시 |
| 심박수 | 분당 심박 횟수 (bpm) |

### 4-2. 혈압 정보

| 필드 | 설명 |
|------|------|
| 회원 ID | 측정 대상 회원 식별자 |
| 측정일시 | 측정이 이루어진 일시 |
| 수축기 혈압 | Systolic Blood Pressure (mmHg) |
| 이완기 혈압 | Diastolic Blood Pressure (mmHg) |
| 혈압 상태 | 정상 / 주의 / 위험 등 상태 코드 |

### 4-3. 체중 정보

| 필드 | 설명 |
|------|------|
| 회원 ID | 측정 대상 회원 식별자 |
| 측정일시 | 측정이 이루어진 일시 |
| 체중 | kg 단위 |
| BMI | Body Mass Index |

### 4-4. 걸음수 정보

| 필드 | 설명 |
|------|------|
| 회원 ID | 측정 대상 회원 식별자 |
| 전송일시 | 데이터가 전송된 일시 |
| 당일 누적 걸음수 | 자정 기준 당일 누계 |

---

## 5. 알림 기능

**요구사항**
- 심박, 혈압, 혈당 데이터 수신 중 이상 수치 감지 시 관리자에게 알림 전송
- 알림 대상: 의사(관리자)
- 이상 데이터 판정 기준은 각 지표별 정상 범위를 벗어난 경우로 정의
