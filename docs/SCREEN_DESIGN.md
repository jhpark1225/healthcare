# Screen Design Guide

> 모든 화면 생성 및 수정 작업은 반드시
> `./docs/DESIGN.md`를 먼저 참조한 뒤 진행한다.
> 화면의 색상, 간격, 폰트, 버튼 스타일, 카드 스타일, 입력폼 스타일, 레이아웃 규칙은
> `DESIGN-apple.md`의 정의를 우선 적용한다.

---

## 공통 규칙

- HTML 샘플은 지시가 있을 때만 생성한다. 지시 전에 먼저 만들지 않는다.
- HTML 생성 지시가 떨어지면 **ReactNative용(모바일)** 과 **React 웹용(웹)** 샘플을 각각 1개씩 만든다.
- 모바일 샘플: 세로형(portrait) 레이아웃
- 웹 샘플: 가로형 레이아웃, 반응형 웹 적용
- HTML 샘플 파일은 `./demo/` 폴더에 저장한다.

### SCR-001 · 로그인

| 항목 | 내용 |
|------|------|
| 화면 ID | SCR-001 |
| 화면명 | 로그인 |
| 사용자 | 의사, 환자 |
| 플랫폼 | React Native (모바일) / React Web (웹) |

#### 목적

의사와 환자가 ID와 Password로 인증하여 시스템에 접근한다.

#### 레이아웃

**모바일 (React Native · 세로형)**
- 화면 중앙 수직 정렬
- 상단: 서비스 로고 / 아이콘
- 중단: ID 입력 필드 → Password 입력 필드
- 하단: 로그인 버튼 → 회원가입 링크

**웹 (React · 가로형 반응형)**
- 뷰포트 중앙 배치, 최대 너비 카드 형태
- 좌측 브랜드 패널 / 우측 폼 패널 (2-column)
- 모바일 브레이크포인트(≤ 480px)에서 단일 컬럼으로 전환

#### 표시 정보

| 요소 | 타입 | 설명 |
|------|------|------|
| 아이디 | Text Input | 사용자 ID 입력 |
| 암호 | Password Input | 비밀번호 입력 (마스킹) |
| 로그인 버튼 | Primary Button | 인증 요청 실행 |
| 회원가입 | Text Link | 회원가입 화면으로 이동 |

#### 디자인 토큰 적용 (DESIGN-apple.md 기준)

| 속성 | 토큰 | 값 |
|------|------|----|
| 페이지 배경 | `canvas-parchment` | `#f5f5f7` |
| 카드 배경 | `canvas` | `#ffffff` |
| 카드 테두리 | `hairline` 1px | `#e0e0e0` |
| 카드 radius | `rounded.lg` | `18px` |
| 입력 radius | `rounded.pill` | `9999px` |
| 입력 높이 | `search-input` 스펙 | `44px` |
| 버튼 스타일 | `button-primary` | pill, `#0066cc`, 높이 44px |
| 버튼 active | 시스템 공통 | `transform: scale(0.95)` |
| 포커스 링 | `primary-focus` 2px solid | `#0071e3` |
| 제목 폰트 | `display-md` | SF Pro Display / Inter, 28px, 600, ls -0.374px |
| 본문 폰트 | `body` | SF Pro Text / Inter, 17px, 400, ls -0.374px |
| 링크 색상 | `primary` | `#0066cc` |
| 그림자 | 없음 | 카드·버튼에 그림자 사용 금지 |
| 그라디언트 | 없음 | 데코레이티브 그라디언트 사용 금지 |

#### 동작 규칙

- ID / Password 둘 다 입력해야 로그인 버튼 활성화
- 인증 실패 시 인라인 에러 메시지 표시 (버튼 하단)
- 에러 텍스트 색상: `#ff3b30` (시스템 레드, Apple HIG 준수)

#### 샘플 파일

| 플랫폼 | 파일 경로 |
|--------|-----------|
| 모바일 (React Native 모의) | `./demo/login-mobile.html` |
| 웹 (React 모의) | `./demo/login-web.html` |

### SCR-002 · 환자 목록

| 항목 | 내용 |
|------|------|
| 화면 ID | SCR-002 |
| 화면명 | 환자 목록 |
| 사용자 | 의사 전용 |
| 플랫폼 | React Native (모바일) / React Web (웹) |

#### 목적

로그인한 의사가 전체 환자 목록을 조회하고 검색하여 환자를 선택한다.
환자 본인이 로그인한 경우 이 화면을 거치지 않고 SCR-003(건강 상세)으로 직접 이동한다.

#### 화면 진입 조건

| 역할 | 진입 경로 |
|------|-----------|
| 의사 | 로그인 성공 → SCR-002 환자 목록 |
| 환자 | 로그인 성공 → SCR-003 건강 상세 (본인) — 목록 화면 skip |

#### 레이아웃

**모바일 (React Native · 세로형)**
- 상단 고정 헤더: 화면 제목("환자 목록") + 로그아웃 아이콘
- 검색 바: 이름 검색 입력 + 성별 필터 칩(전체 / 남 / 여)
- 본문: 환자 카드 리스트 (세로 스크롤)
- 각 카드: 사진(아바타) + 이름 + 성별 + 생년월일 + 우측 꺽쇠(›)

**웹 (React · 가로형 반응형)**
- 좌측 고정 사이드바: 서비스 로고 + 네비게이션 메뉴 + 로그아웃
- 우측 메인 영역:
  - 상단: 페이지 제목 + 검색 바 + 성별 필터 칩
  - 본문: 환자 카드 그리드 (2~3열, 반응형)
- 브레이크포인트 ≤ 768px: 사이드바 → 상단 탭바로 전환, 카드 1열

#### 표시 정보

| 요소 | 타입 | 설명 |
|------|------|------|
| 환자 사진 | Avatar | 프로필 이미지 (없을 경우 이니셜 플레이스홀더) |
| 이름 | Text | 환자 성명 |
| 성별 | Badge | 남 / 여 |
| 생년월일 | Text | YYYY.MM.DD 형식 |
| 검색 입력 | Text Input | 환자 이름 실시간 검색 |
| 성별 필터 | Chip Group | 전체 · 남 · 여 (단일 선택) |

#### 디자인 토큰 적용 (DESIGN-apple.md 기준)

| 속성 | 토큰 | 값 |
|------|------|----|
| 페이지 배경 | `canvas-parchment` | `#f5f5f7` |
| 카드 배경 | `canvas` | `#ffffff` |
| 카드 테두리 | `hairline` 1px | `#e0e0e0` |
| 카드 radius | `rounded.lg` | `18px` |
| 카드 패딩 | `spacing.lg` | `24px` |
| 검색 입력 | `search-input` 스펙 | pill, h44, 17px, rgba(0,0,0,0.08) 테두리 |
| 필터 칩 | `configurator-option-chip` | pill, canvas bg, 14px, padding 12px 16px |
| 필터 칩 선택 | `configurator-option-chip-selected` | 2px solid `primary-focus` |
| 사이드바 배경 (웹) | `surface-tile-1` | `#272729` |
| 사이드바 텍스트 | `on-dark` | `#ffffff` |
| 아바타 플레이스홀더 | `canvas-parchment` | `#f5f5f7` |
| 본문 폰트 | `body` | 17px, 400, ls -0.374px |
| 보조 텍스트 | `ink-muted-48` | `#7a7a7a` |
| 그림자 | 없음 | 카드·버튼에 그림자 사용 금지 |
| 그라디언트 | 없음 | 데코레이티브 그라디언트 사용 금지 |

#### 동작 규칙

- 이름 검색: 입력 즉시 실시간 필터링 (debounce 300ms)
- 성별 필터: 전체 / 남 / 여 중 1개 선택, 기본값 "전체"
- 검색 + 필터 동시 적용 가능
- 환자 카드 클릭 → SCR-003 건강 상세 화면으로 이동
- 검색 결과 없을 때: "검색 결과가 없습니다" 빈 상태 메시지 표시

### SCR-003 · 환자 건강 상세

| 항목 | 내용 |
|------|------|
| 화면 ID | SCR-003 |
| 화면명 | 환자 건강 상세 |
| 사용자 | 의사 (선택한 환자), 환자 (본인) |
| 플랫폼 | React Native (모바일) / React Web (웹) |

#### 목적

환자의 기본 정보와 WebSocket으로 실시간 수신되는 건강 측정값을 그래프로 모니터링한다.
의사는 선택한 환자의 데이터를 조회하고, 환자는 로그인 후 본인 데이터로 직접 진입한다.

#### 화면 진입 조건

| 역할 | 진입 경로 |
|------|-----------|
| 의사 | SCR-002 환자 목록 → 환자 카드 선택 → SCR-003 |
| 환자 | 로그인 성공 → SCR-003 (본인 데이터 자동 연결) |

#### 레이아웃

**모바일 (React Native · 세로형)**
- 상단 고정 헤더: 뒤로가기(의사만) + 화면 제목 + 연결 상태 인디케이터
- 프로필 카드: 아바타 · 이름 · 성별 배지 · 생년월일 · 보유 질환 태그
- 건강 그래프 섹션 (세로 스크롤, 각 항목 카드):
  1. 심박수 (bpm) — 라인 그래프
  2. 혈압 (수축기 / 이완기 mmHg) — 듀얼 라인 그래프
  3. 혈당 (mg/dL) — 라인 그래프 + 상태 배지(normal / elevated / high)
  4. 몸무게 (kg) — 바 그래프
  5. BMI — 바 그래프 + 범위 표시
  6. 골격근량 (kg) — 바 그래프
  7. 체지방률 (%) — 바 그래프

**웹 (React · 가로형 반응형)**
- 좌측 고정 사이드바: SCR-002와 동일 구조 + 현재 환자명 하이라이트
- 우측 메인 영역:
  - 상단 프로필 배너: 아바타 · 이름 · 성별 · 생년월일 · 질환 태그 · 연결 상태
  - 그래프 그리드 (2열):
    - 상단 전체 폭: 심박수 라인 그래프 (가장 빠른 갱신, 4초)
    - 2열 배치: 혈압 / 혈당 / 몸무게 / BMI / 골격근량 / 체지방률
- 브레이크포인트 ≤ 768px: 사이드바 탭바 전환, 그래프 1열

#### 표시 정보

**기본 정보**

| 요소 | 타입 | 설명 |
|------|------|------|
| 아바타 | Avatar | 프로필 이미지 / 이니셜 플레이스홀더 |
| 이름 | Text | 환자 성명 |
| 성별 | Badge | 남 / 여 |
| 생년월일 | Text | YYYY.MM.DD |
| 보유 질환 | Tag Group | 질환 코드 기반 태그 (예: 고혈압, 당뇨병) |
| 연결 상태 | Indicator | WebSocket 연결 중 / 연결 끊김 |

**실시간 건강 그래프**

| 지표 | 단위 | WebSocket 이벤트 | 갱신 주기 | 그래프 유형 |
|------|------|-----------------|----------|-------------|
| 심박수 | bpm | `heartRate` | 4초 | 라인 |
| 수축기 혈압 | mmHg | `bloodPressure` | 2시간 (접속 즉시 1회) | 듀얼 라인 |
| 이완기 혈압 | mmHg | `bloodPressure` | 2시간 (접속 즉시 1회) | 듀얼 라인 |
| 혈당 | mg/dL | `glucose` | 1시간 (접속 즉시 1회) | 라인 + 상태 배지 |
| 몸무게 | kg | `weight` | 하루 3회 08/12/18시 | 바 |
| BMI | — | `weight` | 하루 3회 08/12/18시 | 바 |
| 골격근량 | kg | `weight` | 하루 3회 08/12/18시 | 바 |
| 체지방률 | % | `weight` | 하루 3회 08/12/18시 | 바 |

#### 이상 수치 표시 규칙

| 지표 | 이상 조건 | 표시 방법 |
|------|-----------|-----------|
| 심박수 | `source === "abnormal_event"` 또는 ≥ 100bpm | 그래프 포인트 빨간색 + 상단 알림 배너 |
| 혈당 | `status === "elevated"` (110~139) | 그래프 라인 주황색 |
| 혈당 | `status === "high"` (≥ 140) | 그래프 라인 빨간색 + 알림 배너 |
| 혈압 | 수축기 ≥ 140 또는 이완기 ≥ 90 | 해당 라인 빨간색 + 알림 배너 |

| 그래프 카드 | `store-utility-card` | canvas, hairline 1px, rounded.lg 18px, padding 24px |

#### 동작 규칙

- 화면 진입 시 WebSocket 연결 (`/simulator?userId=...&apiKey=...`) 자동 시작
- 연결 성공 시: `userProfile` 수신 → 기본 정보 렌더 → 각 이벤트 구독 시작
- 각 이벤트 수신마다 해당 그래프에 데이터 포인트 추가 (최대 20개 유지, 초과 시 좌측 제거)
- 이상 수치 감지 시 화면 상단에 알림 배너 표시 (5초 후 자동 해제 또는 수동 닫기)
- 화면 이탈 시 WebSocket 연결 해제
- 연결 끊김 시 인디케이터를 회색으로 변경 + "연결 끊김" 메시지 표시
